Security Policy

Vulnerability reporting and security information

Vulnerability Reporting

We take security seriously and welcome responsible disclosure of security vulnerabilities. If you discover a security issue, please report it to us.

How to Report

  • Email: security@martinlemelle.com
  • Please include detailed information about the vulnerability
  • Provide steps to reproduce the issue
  • Include any relevant screenshots or proof-of-concept code

What We Promise

  • We will acknowledge receipt of your report within 48 hours
  • We will keep you updated on our progress
  • We will credit you in our security acknowledgments (if you wish)
  • We will not take legal action against security researchers acting in good faith

Out of Scope

  • Social engineering attacks
  • Physical security issues
  • Third-party services or applications
  • Denial of service attacks

Note: This website is primarily informational. For security issues related to Grambling State University systems, please contact the university's IT department directly.
